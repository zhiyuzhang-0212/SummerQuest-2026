### 项目与学习经历

#### NEX-N1 后训练：HTML / 前端方向数据

- 时间：2025.08 - 2025.11
- 背景与问题：NEX-N1 后训练阶段中的一个能力子集是前端 / HTML 相关任务。这类样本对数据正确性、可执行性和覆盖面比一般文本任务更苛刻——既要保证生成的页面在结构与语义上能被解析与运行，也要在多轮修改和局部编辑指令下保持一致。
- 个人工作：负责 HTML / 前端方向的后训练数据管道，包括数据来源筛选、清洗与格式规范化、按能力维度（组件构造、样式修改、多轮编辑等）的样本设计与规模评估。
- 方法与结果：在数据管道中补齐了可执行性校验（生成样本能否被解析器无警告加载）和按能力维度的对照抽样，用于指导下一批数据的比例调整；相关数据参与了该版本模型的后训练。
- 局限与反思：受限于样本规模和评测颗粒度，"结构对但视觉细节不对"的样本较难被自动化管道拦下；后续更倾向把渲染截图 + 视觉判分或采样人评引入数据质检。

#### NexAU 运行时：工程与治理

- 时间：2025.12 - 2026.07
- 背景与问题：NexAU 是一个面向多 Agent 编排与 LLM 应用的运行时框架，业务上要同时承载"模型多样、上下文长、图文混合、日志与追踪必须完备"等约束；工程侧要在此之上保证可观测、准确计费、合规拦截和稳定吞吐。
- 个人工作：
  - 设计并落地敏感词治理中间件，支持长文本流式检测、Unicode 归一化（覆盖变长字符导致的越界 bypass）和多语言配置；
  - 修复运行时的多个数据一致性与统计精度问题，包括 usage token 的重复累加、图片 token 低估导致上下文压缩失效、以及带图历史消息的加载与序列化一致性；
  - 参与遥测栈的自建迁移（自托管 PostHog 实例 + 事件模型收敛）和对外统计 API 的建模与实现。
- 方法与结果：坚持"先 RFC、先写用例、再改运行时"的顺序，避免中间件与 per-run 状态耦合导致跨 run 泄漏；每类修复都在多环境 e2e 上做回归验证，改动均已合入并在线上运行。
- 局限与反思：因 Python 与 Rust 运行时并存，早期跨仓库修复的落地周期长于预期；之后更倾向在 RFC 阶段就画清跨仓库依赖图和 bump 顺序，避免"补丁串"式发版。
